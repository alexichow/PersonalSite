Alpha-Beta Pruning Breakthrough Player (A.I.)
Source Code

Breakthrough is a game similar to Checkers with the variation that a piece may move across the board one space either forward or diagonally forward. A piece can only 'capture' an opponents piece in a diagonally-forward move. The first player to reach the other end of the board wins.

My group of three created an artificial intelligence player that used multiple threads, alpha beta pruning, and an evaluation heuristic to determine the next best move. The A.I. player would receive board state information and calculate its next move by using our scoring heuristic. A potential move was scored by evaluating a piece's distance from the end of the board, if it could capture an opposing piece, and the number of remaining pieces for both sides after a capture occurred.

Contribution: I helped develop and implement the evaluation heuristic used by our player in the BreakThroughMove.java file, helped implement the alphaBeta() method of AlphaBetaBreakThroughPlayer.java and implemented the getMove() method of AlphaBetaBreakThroughPlayer.java which initially evaluates the game state for an immediate win condition and launches the threads for evaluating moves if a winning move is not found.


Game of Life
Source Code

Conway's Game of Life is a zero-player "game" that accepts user input for an initial state and then simulates cellular evolution according to the following set of rules:

  1. Any live cell with fewer than two live neighbours dies, as if caused by under-population.
  2. Any live cell with two or three live neighbours lives on to the next generation.
  3. Any live cell with more than three live neighbours dies, as if by overcrowding.
  4. Any dead cell with exactly three live neighbours becomes a live cell, as if by reproduction.

My implementation makes use of Java's UI elements to create and display the board. Each cell is represented by an individual JPanel whose state is updated each generation based on the above rules.

My implementation has the following features:
  • Create an initial state by simply clicking on the 'cells' to mark them for life.
  • Display the current generation.
  • Start, stopping, resuming, and resetting during execution.
  • Taking screenshots (Creates folder for current session to store screenshots)
