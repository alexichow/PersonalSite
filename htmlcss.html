SMITE God Builds
HTML, CSS, Bootstrap, Java, JavaScript, PHP
HTML/CSS/PHP Source Code • Java Source Code • Live Site

SMITE, by Hi-Rez Studios, is an online multiplayer game in which players choose from a wide selection of Gods and compete in 5v5 team matches.

One of the core mechanics of the game is that players spend money accrued throughout the match to purchase items that increase their God's power. Each God is distinctly unique from the others and there are many things to consider when buying items (known as creating a build).
There are currently a few different sites which allow players to create guides for the different Gods, such as smitefire.com and tiermonster.com, but there are none that utilize in-game data to create the most optimal builds.

I decided I wanted to make a site that collected data from the highest level of play, and to use this information to find the most popular build for each character. After I received developer access from Hi-Rez Studios, I decided on the following implementation:

  1. My Java program requests player data from the SMITE API in JSON format and sorts/stores the relevant data from recent matches. For each God, I find the most-bought item for each item slot and create a text file that contains the complete item build for that God (60 total as of now). The text files are then uploaded to the web server.
  2. From the build website, the user can select the god they would like item information for. Using PHP and GET variables, the God they select will pass their name to the actual build page.
  3. The build page receives the God name and opens the relevant text file containing the item build. The page finds each item listed and displays their name and icon on the build page.
I wanted to create a site like this because it uses actual in-game data from professional players to create the builds. These are the players who understand the game at the highest level of play and I felt as though this was a great way to see how they are building their characters.


Myaamia Project Body Parts (Language Learning)
HTML, CSS, JavaScript
Source Code • Demo Site • Live Site

Worked with a small team of four to create a prototype for the Myaamia Project's language learning website.

This project was completed with direct input and feedback from Miami Tribe members at Miami University. The task was to create a website that promoted language learning through three phases:

  1. Observe (Watch and listen to audio playback)
  2. Try (Interact with application and receive feedback)
  3. Test (Test knowledge gained from previous sections)
Our design was selected to be used as the base for the finalized website, which can be viewed here.

Contributions: Helped our main CSS designer, James Fields, develop the CSS for the home page. Developed the Javascript for the 'Try' and 'Test' pages.


'Niiki' iOS App Companion Website
HTML, CSS, JavaScript
iOS app • Live Site

To complement my Senior Design Project group's iOS application created for the Myaamia Project, I was tasked with creating a companion website.

The goal of providing the website in addition to the application was to provide families interested in learning the Miami language with the same resources made available through the app. Not all families have iOS devices, let alone smartphones, so it was important to us to be able to offer them at least some learning materials. The other goal was to host the PDF files which contain the QR codes used by the application in an easily accessible website.

The site features various words and phrases for different room in the house. It utilizes HTML5 audio playback capabilities to play the audio files and supports Internet Explorer, Chrome, Firefox, and Safari web browsers.

More information on our app can be found here: Myaamia Center.
